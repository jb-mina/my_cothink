## 1. 개요

**coThink**는 하나의 질문을 여러 AI 모델에 동시에 보내고, 각 응답을 중재자 모델이 종합·비교해 구조화된 분석을 돌려주는 웹 애플리케이션이다.

**태그라인** — Multi-Model Reasoning Environment (다중 AI 모델 추론 환경)

**UI 언어** — 한국어

## 2. 문제 정의
단일 LLM의 답변은 모델 편향과 지식 누락을 피할 수 없다. 
사용자가 여러 모델을 일일이 열어 같은 질문을 반복 입력하고, 응답을 머릿속에서 대조하는 방식은 비싸고 느리고 일관성이 없다.
coThink는 이 작업을 한 화면 안에 압축한다 — 병렬 질의, 구조화된 종합, 후속 탐구까지.

## 3. 제품 원칙
coThink는 이를 해결하기 위해 다음을 목표로 합니다.
* **관점의 다양성:** 모델 간 공통 의견, 고유 인사이트, 모순점을 한눈에 파악.
* **신뢰성 있는 결론:** 여러 관점을 통합하여 단일 모델보다 객관적인 최선의 답변 도출.
* **탐구형 대화:** 중재자의 후속 질문 제안을 통해 사용자의 사고를 확장.


## 4. 타겟 사용자
- 복잡한 주제에 여러 관점이 필요한 **리서처·애널리스트**
- 모델별 답변 차이를 비교하려는 **AI 파워 유저**
- 비교 분석이 필요한 **의사결정자**
- 다양한 전문가 시각을 탐색하는 **학생·학습자**

## 5. 핵심 기능

### 5.1 다중 모델 동시 질의

같은 질문을 최소 2개 이상의 모델에 병렬로 보낸다. 각 모델은 개별 상태(로딩/완료/에러)로 표시되며, 응답이 도착하는 순서대로 탭에 채워진다.

**현재 지원 모델**

| 모델 | 제공처 |
|---|---|
| GPT-4o | OpenAI |
| Gemini 3 Flash | Google |
| Claude 3.5 Sonnet | Anthropic |
| Perplexity Sonar Pro | Perplexity |
| Grok 4.1 Fast | xAI |

### 5.2 중재자 선택
기본 중재자는 Claude Sonnet 4다. 사용자는 GPT-4o, Gemini 3 Flash, Grok 4.1 Fast, Perplexity Sonar Pro 중에서 대체 중재자를 선택할 수 있다.

### 5.3 구조화된 종합

중재자는 다음 형태로 결과를 돌려준다:

- **Agreement** — 공통된 관점
- **Unique Insights** — 각 모델 고유의 인사이트
- **Contradictions** — 모순 및 이견 (없을 시 "없음")
- **Best Answer** — 종합된 최선의 답변
- **Attributions** — 모델별 기여도 매핑
- **Follow-ups** — 후속 질문 3개

### 5.4 스레드 기반 대화

질문-종합-후속질문의 흐름을 스레드 단위로 묶는다. 사이드바에서 과거 스레드를 열람·삭제하거나 새 스레드를 시작할 수 있으며, 최근 50개가 브라우저에 자동 저장된다. 스레드 제목은 최초 질문의 앞 70자.

### 5.5 후속 질문 플로우

중재자가 제안한 3개의 후속 질문 중 하나를 클릭 한 번으로 재질의한다. 직접 입력도 가능하다. 재질의는 기존 스레드에 새 턴으로 누적되어 맥락이 끊기지 않는다.

### 5.6 인증

단일 공유 비밀번호 기반. 인증되지 않은 방문자는 로그인 페이지로 리디렉션된다. 쿠키는 30일 유지된다.

### 5.7 UI 세부

- 마크다운 렌더링 (볼드, 이탤릭, 인라인/블록 코드, 헤더, 리스트, 표, 구분선)
- 모델별 색상 코딩
- 다크 테마
- Ctrl/Cmd + Enter 단축키로 질의 제출
- 768px 이하에서 사이드바 축소
- 종합 결과 영역 자동 스크롤

## 6. 사용자 플로우

1. URL 방문 → 미인증 시 `/login`으로 리디렉션
2. 비밀번호 입력 → 메인 진입
3. 모델(최소 2개)과 중재자 선택
4. 질문 입력 후 **coThink** 또는 Ctrl/Cmd+Enter
5. **Phase 1** — 선택된 모델들이 병렬로 응답 생성
6. **Phase 2** — 중재자가 응답을 종합 분석
7. **Phase 3** — 구조화된 분석 + 후속 질문 제시, 스레드 자동 저장
8. 후속 질문 클릭 또는 직접 입력으로 턴 추가
9. 사이드바에서 과거 스레드 재개 가능

## 7. 비기능 요구사항
- **지연 시간** — 질의 1회당 일반적으로 5–15초
- **브라우저 호환** — 최신 Chrome / Safari / Firefox
- **반응형** — 데스크톱 / 태블릿 / 모바일
- **보안** — httpOnly·secure 쿠키, 전역 미들웨어 라우트 보호

## 8. 범위 밖 (Out of Scope) / 이후 고려

다음은 **의도적으로** 현재 버전에 포함되지 않는다:

- 다중 사용자 계정 / 개인 워크스페이스
- 서버 측 영구 저장(DB) 및 스레드 공유
- 스레드 내보내기 (Markdown / PDF)
- 스트리밍 응답
- 실패한 모델 호출 자동 재시도
- UI에서의 모델 목록 커스터마이징
- 사용량 / 비용 대시보드
- 이미지·파일 업로드, 음성 입출력
- Rate limiting, 캐싱, 관리자 콘솔

## 9. 성공 지표 (제안)

현재 측정 장치는 없으나 다음을 우선순위로 제안한다:

- **세션당 평균 턴 수** — 후속 질문 활용도 (핵심 지표)
- **후속 질문 클릭률 vs. 직접 입력률** — 중재자 제안 품질의 프록시
- **스레드 재개율** — 사이드바 재방문 비율
- **모델별 에러율** — API 안정성
- **중재자 선택 분포** — 기본값 이탈 여부
