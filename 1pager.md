# coThink — 1pager
---

## 개요

**coThink**는 하나의 질문을 여러 AI 모델에 동시에 보내고, 각 응답을 중재자 모델이 종합·비교해 구조화된 분석을 돌려주는 웹 애플리케이션이다.
**태그라인** — Multi-Model Reasoning Environment (다중 AI 모델 추론 환경)
**UI 언어** — 한국어

## 비전 
AI 도구는 "답을 주는 도구"에서 "판단을 함께 키우는 도구"로 이동해야 한다.
지금 대부분의 AI 제품은 전자에 머물러 있다. 하나의 모델이 하나의 답을 돌려주고, 사용자는 그것을 받아들이거나 다시 묻는다. 하지만 중요한 결정 앞에서 실제로 필요한 건 답이 아니라 관점의 대조, 모순의 발견, 구조화된 판단이다.
coThink는 이 간극을 메우는 첫 번째 실험이다. 장기적으로는 조직 바깥에서 혼자 중요한 결정을 내려야 하는 전문가들을 위한 사고 도구 생태계를 지향하며, coThink는 그 첫 번째 점이다.

## 문제 정의

지난 10년간 혼자 중요한 결정을 내려야 하는 사람이 급격히 늘었다. 솔로 창업자, 1인 전문가, 독립 컨설턴트, 프리랜서 리드. 조직 안에서 동료와의 벽치기로 교정되던 판단 루프가 사라진 자리에, 단일 AI 모델에 기대는 관행이 자리잡았다.
단일 모델은 자기 편향을 검토할 수 없다. 지식이 누락되어도 알려주지 않는다. 다른 모델이 같은 질문에 다르게 답한다는 사실을 사용자가 알더라도, 5개 탭을 띄워 수동으로 대조하는 건 매 결정마다 10분의 노동이다. 대부분의 사람들은 그 노동을 생략하고, 첫 번째 답을 그대로 받아들인다.
coThink는 이 노동을 한 번의 클릭으로 대체한다. 더 나아가 단순 병렬 나열을 넘어서 — 공통점 · 고유 인사이트 · 모순 · 종합된 답이라는 의사결정자의 인지 구조에 맞춘 형태로 재조합한다.

## 제품 관점
세 가지 믿음이 이 제품을 만든다.

a. 답이 아니라 관점의 대조가 필요하다
전문가의 판단은 "정답"이 아니라 "여러 관점 중 최선의 선택"에서 나온다. coThink는 단일한 답을 먼저 주지 않는다. 복수의 관점을 먼저 보여주고, 그다음 구조화한다. 순서가 중요하다.

b. 구조는 해석이다
5개 응답을 나란히 보여주는 것만으로는 부족하다. 사람의 인지 부하가 그대로 남는다. coThink의 중재자는 응답을 공통점 / 고유 인사이트 / 모순 / 종합된 답 네 축으로 분해한다. 이 네 축은 경험 많은 의사결정자가 실제로 정보를 분류하는 방식과 일치한다. 단순 요약이 아니라 해석 프레임워크다.

c. AI는 실행자가 아니라 벽치기 상대다
지금의 AI 에이전트 담론은 "AI가 대신 일을 해준다"에 치우쳐 있다. coThink는 정반대다. 판단의 주체는 사람이고, AI는 그 판단을 단련시키는 파트너다. 후속 질문을 제안하는 기능도 여기서 나온다 — "대신 결정해드릴게요"가 아니라 "이것도 생각해보셨나요?"의 자세.


## 타겟 사용자
특히 아래 사용자들일수록 보다 완성도 있는 결론을 가장 효율적으로 도출할 수 있다.
- 복잡한 주제에 여러 관점이 필요한 **리서처·애널리스트**
- 모델별 답변 차이를 비교해 결정하고 싶은 **AI 파워 유저**
- 면밀한 비교 분석이 반드시 필요한 **의사결정자**
- 다양한 전문가 시각을 탐색하는 **학생·학습자**

## 5. 핵심 기능

### 5.1 다중 모델 동시 질의 및 답변 
같은 질문을 내가 선택한 최소 2개 이상의 모델에게 동시에 보낸다. 
각 모델은 개별 상태(로딩/완료/에러)로 답변을 제공하며, 응답이 도착하는 순서대로 확인할 수 있다.


### 5.2 중재자 선택
기본 중재자를 유저가 원하는대로 설정할 수 있다. 
가령, 사용자는 GPT-4o, Gemini 3 Flash, Grok 4.1 Fast, Perplexity Sonar Pro 중에서 대체 중재자를 선택할 수 있다.

### 5.3 구조화된 종합
중재자는 다음 형태로 결과를 돌려준다:
- **Agreement** — 공통된 관점
- **Unique Insights** — 각 모델 고유의 인사이트
- **Contradictions** — 모순 및 이견 (없을 시 "없음")
- **Best Answer** — 종합된 최선의 답변
- **Attributions** — 모델별 기여도 매핑
- **Follow-ups** — 후속 질문 3개

### 5.4 스레드 기반 대화
질문-종합-후속질문의 흐름을 스레드 단위로 묶는다. 사이드바에서 과거 스레드를 열람·삭제하거나 새 스레드를 시작할 수 있으며, 최근 50개가 브라우저에 자동 저장된다. 스레드 제목은 최초 질문의 앞 70자.

### 5.5 후속 질문 플로우
중재자가 제안한 3개의 후속 질문 중 하나를 클릭 한 번으로 재질의한다. 직접 입력도 가능하다. 재질의는 기존 스레드(질문+답변들) 맥락에서 답변되어 맥락을 유지하며 새 턴으로 누적되어 맥락이 끊기지 않는다.

### 5.6 인증

단일 공유 비밀번호 기반. 인증되지 않은 방문자는 로그인 페이지로 리디렉션된다. 쿠키는 30일 유지된다.

### 5.7 UI 세부

- 마크다운 렌더링 (볼드, 이탤릭, 인라인/블록 코드, 헤더, 리스트, 표, 구분선)
- 모델별 색상 코딩
- 다크 테마
- Ctrl/Cmd + Enter 단축키로 질의 제출
- 768px 이하에서 사이드바 축소
- 종합 결과 영역 자동 스크롤

## 6. 사용자 플로우

1. URL 방문 → 미인증 시 `/login`으로 리디렉션
2. 비밀번호 입력 → 메인 진입
3. 모델(최소 2개)과 중재자 선택
4. 질문 입력 후 **coThink** 또는 Ctrl/Cmd+Enter
5. **Phase 1** — 선택된 모델들이 병렬로 응답 생성
6. **Phase 2** — 중재자가 응답을 종합 분석
7. **Phase 3** — 구조화된 분석 + 후속 질문 제시, 스레드 자동 저장
8. 후속 질문 클릭 또는 직접 입력으로 턴 추가
9. 사이드바에서 과거 스레드 재개 가능

## 7. 비기능 요구사항
- **지연 시간** — 질의 1회당 일반적으로 5–15초
- **브라우저 호환** — 최신 Chrome / Safari / Firefox
- **반응형** — 데스크톱 / 태블릿 / 모바일
- **보안** — httpOnly·secure 쿠키, 전역 미들웨어 라우트 보호

## 8. 범위 밖 (Out of Scope) / 이후 고려

다음은 **의도적으로** 현재 버전에 포함되지 않는다:

- 다중 사용자 계정 / 개인 워크스페이스
- 서버 측 영구 저장(DB) 및 스레드 공유
- 스레드 내보내기 (Markdown / PDF)
- 스트리밍 응답
- 실패한 모델 호출 자동 재시도
- UI에서의 모델 목록 커스터마이징
- 사용량 / 비용 대시보드
- 이미지·파일 업로드, 음성 입출력
- Rate limiting, 캐싱, 관리자 콘솔

## 9. 성공 지표 (제안)

현재 측정 장치는 없으나 다음을 우선순위로 제안한다:

- **세션당 평균 턴 수** — 후속 질문 활용도 (핵심 지표)
- **후속 질문 클릭률 vs. 직접 입력률** — 중재자 제안 품질의 프록시
- **스레드 재개율** — 사이드바 재방문 비율
- **모델별 에러율** — API 안정성
- **중재자 선택 분포** — 기본값 이탈 여부
